## Before You Start / Safety

This guide is for **Mazda MX-5 NB (1998-2005)** only.

OBD-II data is a starting point, not a final diagnosis by itself.

> Warning: Avoid replacing parts from one stored code alone; verify wiring, power/ground, and live conditions first. https://www.obd-codes.com/faq/read-freeze-frame-data.php

> High-risk quote: Clearing codes resets monitor information and can set readiness monitors back to incomplete before inspection. https://www.obdautodoctor.com/tutorials/obd-readiness-monitors-explained/

> Manual-reference quote: Mellens indexes NB-era manuals (1999-2001 and 2005), so connector pinouts, code charts, and model-specific diagnostic trees should be confirmed by year/VIN. https://www.mellens.net/mazda/index.html

## Required Tools

- OBD-II scanner (basic code reader minimum)
- Optional scanner/app with freeze-frame and readiness support
- Notebook or phone notes/screenshots for captured data

## Required Parts / Fluids

- None for scan-only diagnostics

## Model-specific notes (NB1 vs NB2)

### NB1 (1998-2000)
- Some scan apps/readers expose limited live data on older ECUs; verify scanner compatibility before deep diagnosis.

### NB2 (2001-2005)
- Data availability is often better, but monitor behavior still depends on drive-cycle completion and fault status.

> Community quote: r/Miata search repeatedly includes NB posts asking whether 1999/2000 cars are OBD-II and why generic readers show limited data. https://www.reddit.com/r/Miata/search/?q=nb%20obd2%20code&restrict_sr=1

## Step-by-Step Procedure

### 1) Identify symptom and operating context

Record when the issue happens:
- cold start,
- hot restart,
- idle,
- cruise/load,
- refuel events.

Codes without context are easy to misread.

### 2) Pull all code types (if scanner supports)

Read and log:
- stored codes,
- pending codes,
- permanent codes (if present).

Do **not** clear yet.

### 3) Capture freeze-frame data

For each relevant DTC, capture freeze-frame values (RPM, load, coolant temp, speed, trims where available).

> High-risk quote: Freeze-frame shows the operating snapshot when fault criteria were met, helping avoid random parts swapping. https://www.obd-codes.com/faq/read-freeze-frame-data.php

### 4) Check readiness monitor status

Read monitor status before inspection or after repairs.

> High-risk quote: Readiness monitors can be complete, incomplete, disabled, or not available depending on vehicle support and drive-cycle conditions. https://www.obdautodoctor.com/tutorials/obd-readiness-monitors-explained/

### 5) Interpret conservatively

- One pending code ≠ guaranteed bad component.
- Correlate code + freeze-frame + visual checks + known symptom.
- Prioritize power, ground, connector, and vacuum/air leaks before major parts.

### 6) After repairs: clear and re-check

Only clear codes **after** corrective work.

Then drive enough mixed conditions to allow monitors to run, and rescan.

> Community confirmation quote: old.reddit NB OBD2 search context includes repeated reports of “code returns after quick clear” until root cause and monitor cycle completion are addressed. https://old.reddit.com/r/Miata/search?q=nb+obd2+code&restrict_sr=on

## Torque Specs / Capacities (if applicable)

No torque specs apply for scan-only diagnostics.

## Verification / Post-service checks

- Symptom improved/removed in original failure conditions
- Pending/stored codes remain clear after drive cycle
- Readiness monitors appropriate for your inspection/legal requirements

## Uncertainty / Open Questions

- Readiness requirements and allowable “not ready” monitors vary by region/inspection program.
- Community evidence in this run was partly search/snippet level.
- Therefore pass/fail inspection assumptions are explicitly deferred to local rules and year/VIN documentation.
