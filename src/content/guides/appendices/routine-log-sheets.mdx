## Why this matters on NB cars

NBs are now old enough that service history is often partial or inconsistent. A clean log gives you:

- better troubleshooting (what changed, when),
- safer interval planning,
- higher resale confidence,
- fewer repeated jobs from poor handover notes.

If you only do one admin task, do this one.

## Required Tools

- Spreadsheet (Google Sheets/Excel) or printed binder
- Odometer reading at each service
- Phone camera for parts labels/invoices
- Folder/cloud storage for receipts and photos

## One-time setup (recommended columns)

Use these exact columns:

1. Date
2. Odometer (km/mi)
3. Job type (**Baseline / Interval / Corrective / Inspection**)
4. System (engine, cooling, brakes, rust, etc.)
5. Work performed
6. Parts/fluids used (full product name)
7. Spec + quantity
8. Labor notes (DIY/shop)
9. Cost
10. Next due (date + odometer)
11. Evidence link (invoice/photo ID)
12. Result/observations

This structure prevents vague entries like “service done.”

## NB-specific fields worth adding

Add these extra columns for high-value NB ownership:

- Rust check status (sills / rear arches / underbody)
- Convertible drain status (clear / slow / blocked)
- Alignment + tire wear notes (inside edge wear, pull, vibration)
- Cooling behavior notes (fan cycling normal, temp stability)
- Oil consumption (ml per 1,000 km)

## Interval planning table (conservative owner baseline)

Use this as a planning layer, then refine by your usage.

| Item | Conservative check/change baseline | Log trigger to add |
|---|---|---|
| Engine oil + filter | 8,000-10,000 km or 12 months | exact oil grade, filter part no., drain-plug washer used |
| Brake fluid | every 2 years | bleed order + fluid batch/date |
| Coolant | 3-5 years (by coolant type) | coolant chemistry + final mix estimate |
| Manual gearbox oil | 40,000-60,000 km | shift feel before/after |
| Differential oil | 40,000-60,000 km | magnet debris notes |
| Power steering fluid | annual check / flush by condition | pump noise before/after |
| Drain cleaning (soft-top cars) | every 3-6 months | left/right flow status |
| Rust inspection | every 6 months + pre/post winter | photos of known rust-prone points |

## Entry quality rules (simple and strict)

- One completed action = one row.
- Always record **exact** product names/specs from label.
- Every major job needs at least one evidence item (photo or invoice).
- If a task is deferred, create a row with reason and next target.

## Fault-history block (high diagnostic value)

Create a second tab called **Fault History** with:

- first seen date/odometer,
- symptom description,
- trigger conditions,
- tests done,
- resolution,
- recurrence date (if any).

This avoids repeating the same diagnosis every season.

## Monthly review (5-minute routine)

Once per month:

1. Sort by next-due odometer/date.
2. Mark overdue rows in red.
3. Add parts-to-buy list for next month.
4. Check for repeated faults not truly resolved.

## Verification / Post-service checks

- No "unknown last service" items remain
- Every major service has date, odo, exact spec, and evidence
- Next-due plan exists for all interval jobs
- Fault-history tab is up to date
